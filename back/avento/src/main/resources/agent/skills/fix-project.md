# Investiga, corrige e valida um problema dentro do projeto autorizado
Gatilhos: corrigir projeto, corrija projeto, arrumar projeto, ajuste projeto, implementar melhorias projeto

Procedimento:
1. Entenda o pedido e inspecione somente a área relacionada usando directory_tree, search_files e read_file.
2. Identifique a causa antes de editar. Não invente arquivos, stack ou erro.
3. Use edit_file para mudanças pontuais e write_file apenas para arquivo novo ou reescrita necessária.
4. Preserve o estilo e as dependências existentes. Não faça refatorações fora do escopo.
5. Valide com os scripts reais encontrados no projeto usando terminal_run.
6. Confira o resultado das ferramentas antes de afirmar que corrigiu.

As alterações e comandos passam pelo Permission Engine. Se a validação falhar, mostre o erro real e continue corrigindo quando houver uma ação segura e clara.
